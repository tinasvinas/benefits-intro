# 项目固定执行规则

## 新增功能术语统一（强制）

所有表示产生新对象、新记录或新配置的功能，统一使用“新增 XX”作为标准名称。

- 页面入口、按钮文案、弹窗标题、操作提示、Toast、确认文案和需求文档必须使用“新增”。
- 禁止在同一含义下混用“创建”“新建”“添加”等其他说法。
- 函数名和变量名可根据代码语义使用 `create`、`add` 等英文，但用户可见的中文文案必须统一为“新增 XX”。
- 修改涉及相关页面时，应同时检查同一功能链路中的入口、弹窗、校验和成功提示，确保术语一致。

## 需求文档编写与排版规范（强制）

以后新增、改写或重新排版需求文档时，以 `WorkBuddy实训系统需求文档_模板排版确认稿.docx` 的最终确认版本作为格式与结构基准。

- 只记录通用编写规范，不将具体业务需求、业务规则或功能说明写入长期规则。
- 使用 A4 纵向页面及确认稿的字体、字号、页边距、段落间距、表格和列表样式。
- 标题统一使用 Word 内置 `Heading 1` 至 `Heading 6`，允许使用六级标题。
- 标题编号必须使用模板的自动多级编号，不得在标题文字中手工输入章节编号。
- 文档结构按确认稿的模块化组织方式编排；不单独设置“页面入口说明”“数据模型”“UI 要求”“性能需求”等章节。
- 不新增“文档修订说明”“Demo 修订说明”或同类修订记录章节。
- 对现有需求文档重新排版时，只调整结构与版式，不得擅自改变、删减或补充业务内容；用户明确要求删除的章节除外。
- 表格使用模板表格样式并设置明确列宽；列表使用 Word 正式编号或项目符号，不使用手工字符模拟。
- 所有表示产生新对象、新记录或新配置的中文功能术语统一使用“新增”，不得混用“创建”“新建”“添加”。
- 文档完成后必须检查标题层级、自动编号、术语一致性、表格分页和内容完整性；条件允许时应渲染后逐页检查。
