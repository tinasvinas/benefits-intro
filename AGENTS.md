# 项目固定执行规则

## Demo 与需求文档同步（强制）

任何对 Demo 的修改，都必须在同一任务中同步更新 `WorkBuddy实训系统需求文档.md`。

### Demo 修改范围

以下任一文件发生修改，均视为修改 Demo：

- `index.html`
- `pages/**/*.html`
- `assets/shared.js`
- `assets/shared.css`
- 其他直接影响 Demo 页面、交互、文案、数据或流程的资源文件

### 同步要求

完成 Demo 修改后，必须在结束任务前执行以下操作：

1. 更新需求文档首页的“更新日期”。
2. 更新与本次 Demo 变更对应的功能说明、页面流程、字段、校验、权限、状态或交互规则。
3. 在“1.2 主要新增与变更”中补充或调整本次变更记录。
4. 如果本次仅为视觉、文案或缺陷修复且不改变业务规则，也必须在修订说明中记录“实现对齐”，说明需求行为未发生变化。
5. 交付说明中同时列出 Demo 文件和需求文档的更新结果。

`WorkBuddy实训系统需求文档.md` 即使被 `.gitignore` 忽略，也必须在本地同步更新，不得因此跳过。

### 完成门槛

只要 Demo 文件有修改而需求文档没有同步修改，该任务就视为未完成，不得报告“已完成”。
