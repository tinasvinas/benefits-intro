# 初发 Trufar - 前端开发规范

## 1. 弹窗（Modal）规范

### 1.1 按钮文案规范

所有弹窗的底部按钮必须遵循以下统一规范：

#### 标准配置
- **取消按钮**：文案固定为"取消"，使用 `btn-outline` 样式
- **确认按钮**：文案固定为"确定"，使用 `btn-primary` 样式

#### 代码示例
```html
<div class="modal-footer">
  <button class="btn btn-outline" data-close-modal>取消</button>
  <button class="btn btn-primary" onclick="handleSave()">确定</button>
</div>
```

#### 特殊场景

**多步骤弹窗**
- 步骤导航中保留"上一步"、"下一步"按钮
- 最后一步使用"确定"按钮

```html
<div class="modal-footer">
  <button class="btn btn-outline" data-close-modal>取消</button>
  <button class="btn btn-ghost" id="btn-prev">← 上一步</button>
  <button class="btn btn-primary" id="btn-next">下一步 →</button>
  <button class="btn btn-primary" id="btn-save" style="display:none">确定</button>
</div>
```

**查看类弹窗（只读）**
- 仅显示"关闭"按钮

```html
<div class="modal-footer">
  <button class="btn btn-outline" data-close-modal>关闭</button>
</div>
```

#### ❌ 错误示例
```html
<!-- 错误：不要使用其他文案 -->
<button class="btn btn-primary">保存</button>
<button class="btn btn-primary">✓ 保存修改</button>
<button class="btn btn-primary">确认开通</button>
<button class="btn btn-primary">提交</button>
```

#### ✅ 正确示例
```html
<!-- 正确：统一使用"确定" -->
<button class="btn btn-primary">确定</button>
```

### 1.2 按钮顺序规范

按钮从左到右的顺序：
1. **取消/关闭**（灰色 outline 按钮）
2. **次要操作**（如"上一步"，ghost 样式）
3. **主要操作**（如"确定"、"下一步"，primary 样式）

### 1.3 按钮样式规范

| 按钮类型 | 样式类名 | 使用场景 |
|---------|---------|---------|
| 取消/关闭 | `btn btn-outline` | 关闭弹窗，不保存操作 |
| 次要操作 | `btn btn-ghost` | 上一步、其他次要操作 |
| 主要操作 | `btn btn-primary` | 确定、下一步、保存操作 |
| 危险操作 | `btn btn-danger` | 删除、禁用等危险操作 |

## 2. 表单规范

### 2.1 必填标识规范

必填字段使用红色星号标识：

```html
<label><span class="req">*</span>学校名称</label>
```

**注意**：仅在字段确实为必填时才添加星号，避免过度标记。

### 2.2 表单提示文案

使用 `form-hint` 类添加辅助说明：

```html
<div class="form-hint">如 BUPT，将作为登录名前缀</div>
```

## 3. Toast 提示规范

### 3.1 提示类型

```javascript
// 成功提示
toast('操作成功', 'success');

// 警告提示
toast('请输入有效值', 'warning');

// 错误提示
toast('操作失败', 'error');

// 信息提示
toast('未做任何修改', 'info');
```

### 3.2 提示文案规范

- **简洁明确**：直接说明结果，避免冗余
- **正面表达**：使用"已完成"而非"没有失败"
- **具体化**：说明具体操作结果

#### ✅ 正确示例
```javascript
toast('学校信息已保存', 'success');
toast('个人 Token 上限已更新', 'success');
toast('请输入有效的 Token 上限值', 'warning');
```

#### ❌ 错误示例
```javascript
toast('保存成功！', 'success');  // 过于模糊
toast('您的学校信息已经成功保存到系统中', 'success');  // 过于冗长
```

## 4. 命名规范

### 4.1 ID 命名

使用小写字母和连字符（kebab-case）：

```html
<div id="personal-token-limit"></div>
<button id="btn-save-school"></button>
<input id="pkg-start-date">
```

### 4.2 CSS 类命名

使用 BEM 命名法或功能描述：

```html
<div class="modal-overlay"></div>
<div class="tier-card-mini"></div>
<div class="form-group"></div>
```

### 4.3 JavaScript 变量命名

- **全局变量**：大写下划线（SCREAMING_SNAKE_CASE）
- **局部变量**：小驼峰（camelCase）
- **私有变量**：下划线前缀

```javascript
const TIER_INFO = { ... };  // 全局常量
let currentUser = {};       // 局部变量
let _editingTokenLimitKey = null;  // 私有变量
```

## 5. 数据格式规范

### 5.1 日期格式

统一使用 `YYYY-MM-DD` 格式：

```javascript
const startDate = '2026-05-27';
const endDate = '2027-05-26';
```

### 5.2 数值格式

- **Token 数量**：使用数字类型，前端格式化显示
- **金额**：使用分为单位存储，前端转换为元

```javascript
// Token 格式化
function _fmtTokenS(n) {
  if (n >= 100000000) return (n / 100000000).toFixed(2).replace(/\.?0+$/, '') + ' 亿';
  if (n >= 10000) return (n / 10000).toFixed(n >= 1000000 ? 0 : 1).replace(/\.0$/, '') + ' 万';
  return n.toLocaleString();
}
```

### 5.3 布尔值表示

- **无限制**：使用 `-1` 表示
- **启用/禁用**：使用 `true/false`

```javascript
personalToken: -1,  // 无限制
personalToken: 2000000,  // 具体值
customLLM: true,  // 支持
teaching: false,  // 不支持
```

## 6. 注释规范

### 6.1 代码块注释

使用分隔线标注主要功能模块：

```javascript
// ============== 个人 Token 上限设置 ==============
function openPersonalTokenLimitModal(key) {
  // ...
}
```

### 6.2 关键逻辑注释

在复杂逻辑前添加说明：

```javascript
// 根据当前值设置单选框状态
const isUnlimited = d.personalToken === -1;
document.getElementById('ptl-unlimited').checked = isUnlimited;
```

## 7. 交互规范

### 7.1 状态反馈

- 所有异步操作完成后必须提供 Toast 提示
- 表单验证失败时，focus 到错误字段
- 危险操作前必须二次确认

```javascript
if (!confirm('确认删除该增量包？已用部分仍计入消耗记录')) return;
```

### 7.2 禁用状态

已禁用的元素需明确样式：

```css
.disabled {
  opacity: 0.55;
  cursor: not-allowed;
  pointer-events: none;
}
```

### 7.3 加载状态

长时间操作需显示加载提示：

```javascript
showLoading();
await fetchData();
hideLoading();
```

## 8. 可访问性规范

### 8.1 语义化标签

使用正确的 HTML 标签：

```html
<!-- 正确 -->
<button class="btn">操作</button>
<label for="input-name">姓名</label>

<!-- 错误 -->
<div onclick="handleClick()">操作</div>
<span>姓名</span>
```

### 8.2 焦点管理

弹窗打开后，自动聚焦到第一个输入字段：

```javascript
function openModal(id) {
  document.getElementById(id).classList.add('active');
  document.querySelector(`#${id} input:first-of-type`)?.focus();
}
```

## 9. 性能规范

### 9.1 事件委托

列表项的点击事件使用事件委托：

```javascript
// 推荐
document.querySelector('.tier-cards').addEventListener('click', (e) => {
  const card = e.target.closest('.tier-card-mini');
  if (card) handleCardClick(card);
});

// 不推荐
document.querySelectorAll('.tier-card-mini').forEach(card => {
  card.addEventListener('click', () => handleCardClick(card));
});
```

### 9.2 懒加载

大量数据采用懒加载方式：

```javascript
// 仅在展开时才渲染明细
if (!td.dataset.rendered) {
  td.innerHTML = renderHistDetail(recId);
  td.dataset.rendered = '1';
}
```

## 10. Demo 与需求文档同步规范

### 10.1 强制同步条件

任何对 Demo 页面、交互、文案、数据或流程的修改，都必须在同一任务中同步更新 `WorkBuddy实训系统需求文档.md`。

以下文件发生修改时，自动触发需求文档同步：

- `index.html`
- `pages/**/*.html`
- `assets/shared.js`
- `assets/shared.css`
- 其他直接影响 Demo 的资源文件

### 10.2 同步内容

每次修改 Demo 后，至少完成以下更新：

1. 更新需求文档首页的“更新日期”。
2. 更新对应的功能需求、页面流程、字段与校验、权限、状态或交互规则。
3. 在“1.2 主要新增与变更”中补充或调整变更记录。
4. 仅涉及视觉、文案或缺陷修复时，也要记录“实现对齐”，并注明业务规则未改变。
5. 交付时同时说明 Demo 和需求文档的更新结果。

### 10.3 完成门槛

Demo 已修改但需求文档未同步更新时，任务不得判定为完成。需求文档即使被 `.gitignore` 忽略，也不得跳过本地更新。
